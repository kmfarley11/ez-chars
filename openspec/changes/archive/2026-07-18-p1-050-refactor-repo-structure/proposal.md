## Why

The `src/lib/` folder has become a cluttered mix of Svelte components, pure JS utilities, and storage logic. We need to separate seed data, storage loading, and schema definitions from our runtime state variables. This will ensure clean boundaries and significantly speed up codebase navigation for developers and AI agents, building on the normalized data foundation recently established by `p1-060`.

## What Changes

- Create `src/lib/components/` and move all visual `.svelte` files there.
- Create `src/lib/storage/`, move `characterStorage.ts` there, and relocate `src/data.ts` to `src/lib/storage/store.ts`.
- Create `src/lib/utils/` and move pure JS helper utilities (e.g., `urlHelpers.ts`, `theme.ts`, `displayHelpers.ts`, `stringFormatters.ts`) there.
- Add `kit.alias` configurations to `svelte.config.js` (`$components`, `$storage`, `$utils`, `$fixtures`) to prevent fragile relative imports.
- Update all internal imports across the codebase to use these new aliases.
- _Excluded_: 5e page-specific logic (`sheetProjections.ts`, `sheetEditIntents.ts`) will remain feature-local in `src/routes/charsheets/5e/` as its established boundary is already cohesive.
