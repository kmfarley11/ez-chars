## Why

Transition the repository from its legacy Markdown planning/backlog process to OpenSpec as the preferred steady-state engineering workflow. This provides clean context isolation, consistent agent boundaries, and explicit planning checkpoints for all future engineering work.

## What Changes

- **Adopt OpenSpec**: Establish OpenSpec as the primary workflow for active, non-trivial engineering changes.
- **Clarify Agent Boundaries & Artifacts**: Define clear role expectations for agents and establish strict boundaries for planning artifacts to prevent implementation details from bleeding into proposals or specifications.
- **Refine Prioritization Model**: Position the MVP backlog as a prioritized candidate queue rather than an active execution check list.
- **Update Maintainer Guidance**: Document the complete human-facing engineering workflow (including prototyping and reconciliation practices) for maintainer awareness.

## Capabilities

### New Capabilities
- `developer-workflow`: The repository's standard developer and agent workflow rules and gates.

### Modified Capabilities
- None.

## Impact

- **Affected Areas**: Repository guidelines, developer documentation, and backlog management workflows.
